# 就活手帳 機能仕様書

## 1. 概要

就活手帳は、就職活動における企業ごとの選考状況、締切タスク、エントリーシート（ES）、面接記録を一元管理するためのWebアプリケーションである。サーバーやアカウント登録を必要とせず、`index.html` という単一のHTMLファイルのみで動作する。ブラウザ内のlocalStorageにすべてのデータを保存するため、インターネット接続がない環境でも利用でき、個人情報が外部のサーバーへ送信されることもない。任意でFirebaseを用いたクラウド同期を設定すれば、スマートフォンとパソコンなど複数端末間でデータを共有することもできる。

公開版は GitHub Pages（`https://frmcle-m.github.io/shukatsu-app/`）でホストされており、URLを開くだけで利用を開始できる。

## 2. 動作環境

対応ブラウザはモダンブラウザ全般（Chrome、Safari、Edge、Firefoxなど）で、スマートフォン・タブレット・パソコンいずれでも利用できる。画面の縦横比に応じてレイアウトが自動的に切り替わり、横長の画面（画面幅720px以上かつ横長）ではPC向けの2カラムレイアウト、それ以外ではスマートフォン向けのタブ切り替えレイアウトで表示される。このレイアウトは設定画面から「自動／スマホ版／PC版」に固定することもできる。

## 3. 画面構成

アプリは4つの画面と、操作のたびに下からせり上がる形式のモーダル（追加・編集フォームなど）で構成されている。

| 画面 | 役割 |
|---|---|
| タスク（画面A） | 月間カレンダーと、直近の予定・タスク・完了済みタスクの一覧を表示するホーム画面 |
| 企業一覧（画面B） | 登録した応募先企業をカード形式で一覧表示し、検索・絞り込み・並び替えを行う |
| 企業詳細 | 個別の企業について、選考状況・タスク・ES・面接記録をまとめて閲覧・編集する |
| 記録一覧（画面C） | 全企業分のES記録・面接記録を横断して閲覧するための一覧 |

PC版レイアウトでは、常時表示される左カラムにカレンダーが固定され、右カラムに上記の各画面が切り替わって表示される。スマートフォン版では画面下部のタブバー（タスク／企業／記録）で画面を切り替える。

### 3.1 タスク画面（画面A）

月間カレンダーと、その月の予定・タスクを整理して表示する画面。カレンダーには締切のみのタスクが点（ドット）として、期間のあるタスクや予定が横棒（バー）として表示され、同じ週に複数の予定が重なる場合は自動的に段（レーン）に振り分けられる。カレンダーの日付をタップすると、その日のタスク・予定の一覧がポップアップ（PC版では左カラム下に）表示され、そこからタスクの編集や該当企業ページへの移動ができる。年月の見出しをタップすると年月を直接選択するピッカーが開き、当月へ素早く戻るボタンも用意されている。

カレンダーの下には「予定」「タスク」「終了済み」の3つのセクションがあり、企業ごとの絞り込みができるプルダウンも備える。タスクの完了はチェックボックスのワンタップで切り替えられ、締切が2日以内に迫ったタスクは赤字で強調表示される。

### 3.2 企業一覧画面（画面B）

登録済みの企業をカード形式で一覧表示する画面。各カードには企業名、志望度（★1〜5）、業種バッジ、選考状況バッジ、選考結果バッジ、マイページID（登録があればワンタップでコピー可能）が表示される。画面右上の検索アイコンから企業名でのインクリメンタル検索ができるほか、「志望度」「選考状況」による絞り込みモーダル、「50音順／志望度が高い順／選考状況が進んでいる順」の並び替えが利用できる。画面上部の「インターン／本選考」の切り替えにより、同じ企業について2系統の選考トラックを別々に管理できる。画面下部の「企業を追加」ボタンから新規登録が行える。

### 3.3 企業詳細画面

個別企業の情報を管理する画面。上部のヒーロー領域に企業名・業種・志望度・マイページIDを表示し、その下でインターン／本選考の選考トラックを切り替えながら、選考状況（未着手〜内定など）、選考結果（結果待ち・書類落ちなど）、自由記述メモ（応募日程やコースなど）を編集できる。選考状況・選考結果の選択肢は設定画面から自由に追加・並び替え・削除ができる。

画面下部には、その企業に紐づくタスク一覧、ESの記録一覧、面接の記録一覧がそれぞれ表示され、各セクションの「追加」ボタンから新規記録を作成できる。企業情報自体の編集・削除は画面右上の歯車アイコンから行う（削除時は所属するタスク・ES・面接記録もすべて削除される旨の確認ダイアログが表示される）。

### 3.4 記録一覧画面（画面C）

全企業のES記録・面接記録を横断して閲覧するための画面。上部のタブで「ES」と「面接記録」を切り替える。ES一覧では通過可否（すべて／通過／未通過）による絞り込み、ジャンル別・企業別の複数選択絞り込み、「すべて表示／ジャンル別／企業別」のグルーピング表示が可能。面接記録一覧では企業別・日付順の並び替えと、企業による絞り込みができる。

## 4. データモデル

アプリが扱う主なデータ構造は次の通りである。

**企業（Company）**：ID、企業名、業種、志望度（0〜5の整数）、マイページID、インターン／本選考それぞれの選考トラック（選考状況・選考結果・メモを持つ）、その企業に属するタスク配列・ES配列・面接記録配列を持つ。

**タスク／予定（Task）**：ID、名称、種類（タスク／予定）、開始日時（任意）、終了日時（タスクの場合は締切、予定の場合は開催日時）、メモ、リマインド設定（なし／1時間前／3時間前／1日前／3日前／1週間前）、完了フラグを持つ。開始日時を入力すると期間タスク・期間予定として扱われる。

**ES記録**：ID、タイトル、通過可否（真偽値）、提出日、ジャンル（複数選択可）、文字数制限、本文を持つ。本文は全角・半角を区別した文字数カウント（半角は0.5文字換算）が自動表示され、制限文字数を超えると赤字で警告される。

**面接記録**：ID、タイトル、面接日、通過可否（通過／通過待ち／未通過の3択）、次の選考プロセス、メモ・記録本文、音声ファイル（約1.5MBまではBase64形式でブラウザ内に保存し再生可能、それ以上のサイズはファイル名のみ記録）を持つ。

**設定（Settings）**：カレンダーの週の始まり（日曜／月曜）、既定の表示トラック（インターン／本選考）、表示モード（自動／スマホ版／PC版）、文字サイズ（小／中／大）、選考状況・選考結果・業種・ESジャンルそれぞれの選択肢リスト（ユーザーが追加・並び替え・削除可能。ただし初期状態で用意された既定項目は削除できない）を持つ。

## 5. 主な機能

### 5.1 カレンダー連携（.ics書き出し）

タスク・予定の編集フォームから「標準カレンダーに追加」ボタンを押すと、そのタスクの内容をiCalendar形式（.ics）ファイルとして書き出し、iPhone標準カレンダーやGoogleカレンダーなど外部のカレンダーアプリに取り込める。リマインド設定に応じて通知（VALARM）も付与される。iOS端末ではファイルダウンロードの代わりにdata URLでの直接オープンに対応している。

### 5.2 クラウド同期（Firebase連携）

設定画面の「クラウド同期の設定」から、ユーザー自身が用意したFirebaseプロジェクトの設定情報（firebaseConfig）と、端末間で共有する合言葉コードを入力すると、Firestoreを介してリアルタイムにデータを同期できる。匿名認証を使用し、合言葉コードをドキュメントIDとして利用することで、コードを知る端末同士のみがデータへアクセスできる仕組みになっている。同時編集には対応しておらず、後から保存した内容が上書きされる。容量制限を回避するため、面接の音声データは同期対象から除外され、各端末のローカルストレージにのみ保持される。初回接続時は、クラウド側に既存データがある場合、取り込むか上書きするかをユーザーが選択できる。

### 5.3 データ保存方式

すべてのデータは端末のブラウザのlocalStorageに自動保存される（入力から約250ミリ秒後にデバウンス保存）。初回起動時にはデータが存在しないため、操作方法を把握しやすいようにサンプル企業3社分のデモデータが自動投入される。アプリを閉じる・タブを非表示にするタイミングでも、未保存の変更を確実に書き込む処理が入っている。

### 5.4 レイアウト・表示のカスタマイズ

設定画面から、カレンダーの週始まり（日曜／月曜）、表示モード（自動／スマホ版／PC版固定）、インターン・本選考のどちらを既定表示にするか、文字サイズ（小・中・大）を変更できる。表示モードが「自動」の場合、ウィンドウのリサイズに追従してスマホ版・PC版が自動的に切り替わる。

### 5.5 選択肢のカスタマイズ

選考状況、選考結果、業種、ESジャンルの各選択肢は、設定画面の「項目の編集」からユーザーが自由に追加・並び替え（上下ボタン）・削除できる。初期状態で用意されている既定の選択肢（未着手〜内定、結果待ち〜内定蹴り、IT・通信〜その他、企業志望理由〜その他）は誤操作防止のため削除できないようになっている。

## 6. プライバシーとセキュリティに関する留意点

本アプリはユーザーが入力したデータ（企業名、選考状況、ES本文、面接メモ、音声ファイルなど）を外部サーバーへ送信しない設計になっている。クラウド同期機能を有効にした場合のみ、ユーザー自身が設定したFirebaseプロジェクトへデータが送信される。アプリのソースコード自体にはAPIキーや個人情報は含まれておらず、GitHub上で公開しても安全である。ただし、公開リポジトリであるため、ソースコードの内容は誰でも閲覧できる状態にある。

## 7. 制限事項

クラウド同期は同時編集を検知できず、最後に保存した端末の内容で上書きされる。面接の音声ファイルは約1.5MBを超えるとファイル名のみが記録され、実際の音声データは保存されない。データのエクスポート・インポート機能（他ブラウザへの手動移行手段）は現バージョンには実装されていない。
